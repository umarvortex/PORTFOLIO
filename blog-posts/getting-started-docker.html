Getting Started with Docker

Docker has revolutionized how developers build, ship, and run applications by introducing containerization to the mainstream. This beginner's guide will walk you through the fundamental concepts of Docker and help you get started with containerizing your web development projects.

Whether you're a frontend developer looking to standardize your development environment or a backend developer wanting to simplify deployment, Docker offers powerful solutions that can streamline your workflow.

What is Docker?

Docker is an open-source platform that automates the deployment of applications inside lightweight, portable containers. A container is a standard unit of software that packages code and all its dependencies so the application runs quickly and reliably from one computing environment to another.

Key Benefits of Docker

  • Consistency: Eliminates "it works on my machine" problems by providing the same environment everywhere
  • Isolation: Containers run independently without interfering with each other
  • Portability: Runs the same regardless of host environment (Windows, macOS, Linux)
  • Efficiency: Uses fewer resources than virtual machines as containers share the host's OS kernel
  • Rapid Deployment: Containers can be created, replicated, and destroyed in seconds

Understanding Docker Architecture

Before diving into commands, let's understand the core components of Docker:

  • Docker Engine: The client-server application that builds and runs containers
  • Docker Client: The command-line interface used to interact with Docker
  • Docker Daemon: A background service that manages Docker objects
  • Docker Images: Read-only templates containing instructions for creating containers
  • Docker Containers: Runnable instances created from Docker images
  • Docker Registry: A repository for Docker images (like Docker Hub)

Installing Docker

Let's start by installing Docker on your system. Docker provides Docker Desktop for Windows and macOS, which includes Docker Engine, Docker CLI, Docker Compose, and other tools.

For Windows

  1. Visit Docker Desktop for Windows
  2. Download the installer and run it
  3. Follow the installation wizard (Windows 10/11 Pro, Enterprise, or Education with Hyper-V enabled is recommended)
  4. For Windows Home, Docker Desktop will use WSL 2 instead

For macOS

  1. Visit Docker Desktop for Mac
  2. Download the .dmg file for your CPU architecture (Intel or Apple Silicon)
  3. Open the installer and drag Docker to your Applications folder

For Linux

For Ubuntu, you can install Docker Engine with these commands:

sudo apt-get update
sudo apt-get install apt-transport-https ca-certificates curl software-properties-common
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io
sudo usermod -aG docker $USER

Note: After installing, you may need to restart your computer. To verify your installation, open a terminal and run docker --version and docker-compose --version.

Docker Basics: Essential Commands

Working with Images

Images are the blueprints for containers. Here's how to work with them:

  • Pull an image from Docker Hub:
    docker pull nginx
  • List all images:
    docker images
  • Remove an image:
    docker rmi nginx
  • Search for images on Docker Hub:
    docker search node

Working with Containers

Once you have images, you can create and manage containers:

  • Run a container:
    docker run --name my-nginx -p 8080:80 -d nginx
  • List running containers:
    docker ps
  • List all containers (including stopped):
    docker ps -a
  • Stop a container:
    docker stop my-nginx
  • Start a stopped container:
    docker start my-nginx
  • Remove a container:
    docker rm my-nginx
  • View container logs:
    docker logs my-nginx
  • Execute a command in a running container:
    docker exec -it my-nginx bash

Creating Your First Dockerfile

A Dockerfile is a text document containing instructions to build a Docker image. Let's create a simple Dockerfile for a Node.js application: 

# Use an official Node.js runtime as the base image
FROM node:14

# Set the working directory
WORKDIR /usr/src/app

# Copy package.json and package-lock.json
COPY package*.json ./

# Install dependencies
RUN npm install

# Copy application code
COPY . .

# Expose the port the app runs on
EXPOSE 3000

# Command to run the application
CMD ["node", "app.js"]

To build an image from this Dockerfile:

docker build -t my-node-app .

And to run it:

docker run -p 3000:3000 -d my-node-app

Docker Compose for Multi-Container Applications

Docker Compose simplifies managing multi-container applications. Create a docker-compose.yml file to define services, networks, and volumes: 

version: '3'
services:
  web:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - ./:/usr/src/app
    environment:
      - NODE_ENV=development
    depends_on:
      - db
  db:
    image: mongo:4.4
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db

volumes:
  mongodb_data:

Run your multi-container application with:

docker-compose up -d

And stop it with:

docker-compose down

Practical Example: Dockerizing a Web Application

Let's walk through a complete example of containerizing a simple web application with a frontend, backend, and database.

Project Structure

my-app/
├── frontend/
│   ├── Dockerfile
│   ├── package.json
│   └── src/
├── backend/
│   ├── Dockerfile
│   ├── package.json
│   └── src/
├── docker-compose.yml
└── README.md

Frontend Dockerfile (React)

# Build stage
FROM node:14 AS build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# Production stage
FROM nginx:alpine
COPY --from=build /app/build /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

Backend Dockerfile (Node.js/Express)

FROM node:14
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 4000
CMD ["npm", "start"]

Docker Compose File

version: '3'
services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    
  backend:
    build: ./backend
    ports:
      - "4000:4000"
    environment:
      - DATABASE_URL=mongodb://db:27017/myapp
    depends_on:
      - db
    
  db:
    image: mongo:4.4
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db
    environment:
      - MONGO_INITDB_DATABASE=myapp

volumes:
  mongodb_data:

Docker Best Practices

  • Use official images when possible as a starting point
  • Keep images small by using Alpine-based images and multi-stage builds
  • Don't run containers as root when possible
  • Use specific image tags rather than 'latest' for predictability
  • Leverage build cache by ordering Dockerfile commands efficiently
  • Set resource limits on containers to prevent resource exhaustion
  • Use volumes for persistent data
  • Use environment variables for configuration

Troubleshooting Common Docker Issues

Container Won't Start

Check the logs: docker logs [container_id]

Port Conflicts

If you see "port is already allocated" errors, change the host port in your run command: docker run -p 8081:80 nginx

Permission Issues

On Linux, you might need to use sudo or add your user to the docker group: sudo usermod -aG docker $USER

Disk Space Issues

Remove unused containers, images, and volumes: docker system prune -a

Conclusion

Docker has transformed how we develop, test, and deploy applications. By containerizing your applications, you gain consistency, portability, and efficiency. This guide has covered the basics to get you started, but Docker offers much more, including orchestration with Kubernetes, Docker Swarm, advanced networking, and security features.

As you continue your Docker journey, consider exploring Docker Compose for more complex applications, Docker Swarm or Kubernetes for orchestration, and CI/CD pipelines with Docker to automate your build and deployment processes.

Pro Tip: Start small by containerizing a simple application, then gradually move to more complex setups as you gain confidence with Docker concepts and commands.
