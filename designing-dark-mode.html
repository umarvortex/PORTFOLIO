Designing for Dark Mode

Dark mode has evolved from a developer-centric feature to a mainstream design trend that users have come to expect in modern applications. Beyond its aesthetic appeal, dark mode offers practical benefits like reduced eye strain in low-light conditions, extended battery life on OLED screens, and improved accessibility for users with certain visual impairments.

In this article, we'll explore best practices and considerations when implementing dark mode in your web and mobile applications, ensuring a seamless user experience regardless of their preferred theme.

Why Dark Mode Matters

The surge in dark mode adoption isn't just a passing trend. Here are key reasons why it's become essential for modern digital products:

  • Reduced Eye Strain: Dark interfaces emit less light, which can be more comfortable for extended usage, especially at night.
  • Better Battery Life: On devices with OLED or AMOLED displays, dark pixels use significantly less power than bright ones.
  • Accessibility Benefits: Some users with light sensitivity or certain visual impairments find dark interfaces more comfortable.
  • Aesthetic Appeal: Many users simply prefer the sleek, modern look of dark interfaces.
  • User Expectation: Major platforms and apps now support dark mode, making it an expected feature.

Color Theory for Dark Mode

Successfully designing for dark mode requires more than just inverting colors or making everything black. Understanding color relationships in dark contexts is crucial.

Color Perception in Dark Environments

Human vision works differently in dark environments, with a few important considerations:

  • Halation Effect: Light text on dark backgrounds can appear to "bleed" for some users, making it seem blurry.
  • Contrast Sensitivity: The perceived contrast between colors can shift in dark contexts.
  • Color Vibration: Highly saturated colors can appear to "vibrate" against very dark backgrounds, causing visual discomfort.

Creating Effective Dark Mode Color Palettes

When designing a dark color scheme, follow these principles:

  • Avoid Pure Black: Instead of #000000, use soft blacks like #121212 or #1F1F1F to reduce contrast and eye strain.
  • Reduce Color Saturation: Highly saturated colors can be jarring on dark backgrounds. Decrease saturation by 10-30%.
  • Increase Brightness: Colors often need to be brighter on dark backgrounds to maintain the same visual hierarchy.
  • Maintain Sufficient Contrast: Ensure text remains readable with a minimum contrast ratio of 4.5:1 for normal text (WCAG AA standard).

Implementation Strategies

There are several approaches to implementing dark mode in web applications, each with its own advantages:

CSS Custom Properties (Variables)

The most flexible approach is to use CSS variables to define your color scheme: 

:root {
  /* Light theme (default) */
  --background: #ffffff;
  --surface: #f5f5f5;
  --text-primary: #333333;
  --text-secondary: #757575;
  --accent: #5c6bc0;
}

@media (prefers-color-scheme: dark) {
  :root {
    /* Dark theme */
    --background: #121212;
    --surface: #1e1e1e;
    --text-primary: #e4e4e4;
    --text-secondary: #a0a0a0;
    --accent: #7986cb;
  }
}

This approach automatically applies the dark theme when the user has their system set to dark mode. Your components then reference these variables: 

body {
  background-color: var(--background);
  color: var(--text-primary);
}

.card {
  background-color: var(--surface);
  color: var(--text-primary);
}

Manual Theme Toggle

To allow users to manually switch between themes, use a class-based approach: 

:root {
  /* Light theme variables */
}

.dark-theme {
  /* Dark theme variables */
}

/* JavaScript to toggle the class */
document.getElementById('themeToggle').addEventListener('click', () => {
  document.body.classList.toggle('dark-theme');
  // Save preference to localStorage
  const isDark = document.body.classList.contains('dark-theme');
  localStorage.setItem('darkTheme', isDark);
});

Combining System Preference and User Choice

The ideal approach is to respect both system preferences and user choices: 

// Check for saved preference first
const savedTheme = localStorage.getItem('darkTheme');

if (savedTheme !== null) {
  // User has explicitly set a preference
  document.body.classList.toggle('dark-theme', savedTheme === 'true');
} else {
  // No saved preference, use system setting
  const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
  document.body.classList.toggle('dark-theme', prefersDark);
}

// Listen for system changes
window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', (e) => {
  // Only apply if user hasn't set a preference
  if (localStorage.getItem('darkTheme') === null) {
    document.body.classList.toggle('dark-theme', e.matches);
  }
});

Design Considerations for Dark Mode

Beyond basic color swapping, dark mode requires specific design considerations:

Depth and Elevation

In light modes, shadows create depth. In dark modes, both shadows and subtle lighter backgrounds help indicate elevation: 

.card {
  background-color: var(--surface);
}

.dark-theme .card {
  /* Subtle lighter background for elevated elements */
  background-color: rgba(255, 255, 255, 0.05);
}

Images and Media

Consider how images with transparent backgrounds or predominantly light content appear in dark contexts:

  • Invert UI Icons: Icons should be light on dark backgrounds
  • Add Subtle Borders: For light images in dark contexts, add a subtle border
  • Reduce Brightness/Contrast: Consider dimming very bright images in dark mode
@media (prefers-color-scheme: dark) {
  .icon:not(.preserve-color) {
    filter: invert(1);
  }
  
  .content-image {
    border: 1px solid rgba(255, 255, 255, 0.1);
    opacity: 0.85; /* Slightly reduce brightness */
  }
}

State Indicators

Active states, hover effects, and focus indicators may need adjustment in dark mode: 

/* Light mode */
.button:hover {
  background-color: rgba(0, 0, 0, 0.05);
}

/* Dark mode */
.dark-theme .button:hover {
  background-color: rgba(255, 255, 255, 0.1);
}

Testing Dark Mode Designs

Thorough testing is essential for effective dark mode implementations:

  • Contrast Checking: Use tools like the WebAIM Contrast Checker to verify accessibility
  • Device Testing: Test on actual devices, especially those with OLED displays
  • Accessibility Testing: Verify that screen readers and other assistive technologies work properly
  • User Preference Testing: Test that user theme preferences are saved and applied correctly
  • Real Environment Testing: Test both themes in appropriate lighting conditions

Common Pitfalls to Avoid

  • Pure Black (#000000): Too much contrast can cause eye strain and the halation effect
  • Direct Color Inversion: Simply inverting colors often produces poor results
  • Forgetting About Images: Not adjusting how images display in dark contexts
  • Ignoring Visual Hierarchy: Not preserving the same hierarchy between light and dark modes
  • Overlooking Transitions: Jarring switches between themes without smooth transitions

Dark Mode in Popular Frameworks

React

// Using React Context for theme management
const ThemeContext = React.createContext({
  isDark: false,
  toggleTheme: () => {}
});

function ThemeProvider({ children }) {
  const [isDark, setIsDark] = useState(
    window.matchMedia('(prefers-color-scheme: dark)').matches
  );
  
  const toggleTheme = () => {
    setIsDark(!isDark);
    document.body.classList.toggle('dark-theme', !isDark);
  };
  
  useEffect(() => {
    document.body.classList.toggle('dark-theme', isDark);
  }, [isDark]);
  
  return (
    
      {children}
    
  );
}

Conclusion

Dark mode is no longer optional for modern digital products. By understanding the principles of color in dark environments, implementing thoughtful theme switching, and testing thoroughly, you can create dark mode experiences that delight users and enhance accessibility.

Remember that dark mode isn't just an aesthetic choice—it's about respecting user preferences, improving usability in different lighting conditions, and designing with inclusivity in mind.
