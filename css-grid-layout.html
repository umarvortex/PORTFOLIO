Introduction to CSS Grid Layout

CSS Grid Layout has revolutionized how we design website layouts. It provides a two-dimensional layout system, working with both rows and columns, that enables developers to create complex and responsive web layouts with relatively simple and clean HTML and CSS.

What is CSS Grid Layout?

CSS Grid Layout (also known as "Grid") is a two-dimensional layout system designed specifically for the web. It allows you to organize content into rows and columns, and has many features that make building complex layouts straightforward.

Unlike Flexbox, which is primarily designed for one-dimensional layouts (either rows OR columns), Grid excels at dividing a page into major regions and defining the relationship in terms of size, position, and layer between parts of a control built from HTML primitives.

Basic Terminology

Before we dive into examples, let's understand some key terminology:

  • Grid Container: The element that becomes a grid layout when you set its display property to grid or inline-grid.
  • Grid Item: Any direct child of a grid container.
  • Grid Line: The dividing lines that make up the structure of the grid, which can be vertical ("column grid lines") or horizontal ("row grid lines").
  • Grid Track: The space between two adjacent grid lines, either a row or a column.
  • Grid Cell: The intersection of a row and a column.
  • Grid Area: The total space surrounded by four grid lines; it may contain multiple grid cells.

Creating Your First Grid

To create a grid container, set the display property to "grid" on an element: 

.container {
  display: grid;
}

Once you've established a grid container, you can define the columns and rows using the grid-template-columns and grid-template-rows properties: 

.container {
  display: grid;
  grid-template-columns: 200px 200px 200px;
  grid-template-rows: 100px 100px;
  gap: 10px;
}

This creates a grid with three columns, each 200px wide, and two rows, each 100px tall, with a 10px gap between each cell.

The fr Unit

CSS Grid introduces the fr unit, which represents a fraction of the available space. It simplifies creating flexible grid layouts: 

.container {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  gap: 10px;
}

This creates a three-column grid where the middle column takes up twice as much space as the first and third columns.

Responsive Grids with minmax() and repeat()

The minmax() function and repeat() notation make creating responsive grids much easier: 

.container {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 20px;
}

This creates a responsive grid where each column is at least 250px wide and will expand to fill additional space. The grid will automatically fit as many columns as possible into the container.

There are two variants for this pattern:

  • auto-fill: Creates as many tracks as possible, even empty ones
  • auto-fit: Fits the currently available items into the space, collapsing any empty tracks

Positioning Items on the Grid

Grid items can be precisely positioned using the following properties:

  • grid-column-start and grid-column-end: Specify which column lines the item should start and end at
  • grid-row-start and grid-row-end: Specify which row lines the item should start and end at
  • grid-column and grid-row: Shorthand properties for the above
  • grid-area: An even shorter shorthand for all four properties
.item {
  grid-column: 1 / 3;  /* Start at column line 1, end at column line 3 */
  grid-row: 2 / 4;     /* Start at row line 2, end at row line 4 */
}

/* Or, using grid-area */
.item {
  grid-area: 2 / 1 / 4 / 3;  /* row-start / column-start / row-end / column-end */
}

You can also use the span keyword to specify how many tracks an item should span: 

.item {
  grid-column: 1 / span 2;  /* Start at line 1 and span 2 columns */
  grid-row: 2 / span 2;     /* Start at line 2 and span 2 rows */
}

Named Grid Areas

One of the most powerful features of CSS Grid is the ability to name grid areas and place items in them using the grid-template-areas property: 

.container {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-template-rows: auto auto auto;
  grid-template-areas:
    "header header header header"
    "sidebar main main main"
    "footer footer footer footer";
  gap: 10px;
}

.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.footer { grid-area: footer; }

This creates a classic website layout with a header, sidebar, main content area, and footer. The visual representation in the CSS makes the layout very clear and easy to understand.

Alignment in CSS Grid

Grid provides powerful alignment capabilities:

Align Items within Their Cells

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  align-items: center;      /* Vertical alignment (stretch, start, end, center) */
  justify-items: center;    /* Horizontal alignment (stretch, start, end, center) */
}

Align the Grid Within the Container

.container {
  display: grid;
  grid-template-columns: repeat(3, 100px);
  grid-template-rows: repeat(2, 100px);
  height: 400px;
  align-content: center;      /* Vertical alignment of the entire grid */
  justify-content: center;    /* Horizontal alignment of the entire grid */
}

Responsive Layouts with Grid

CSS Grid makes it easy to create responsive layouts without media queries (although they can still be useful for major layout changes): 

.container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 1rem;
}

For more complex responsive layouts, you can combine Grid with media queries: 

.container {
  display: grid;
  grid-template-columns: 1fr;
  grid-template-areas:
    "header"
    "main"
    "sidebar"
    "footer";
  gap: 10px;
}

@media (min-width: 768px) {
  .container {
    grid-template-columns: 200px 1fr;
    grid-template-areas:
      "header header"
      "sidebar main"
      "footer footer";
  }
}

@media (min-width: 1024px) {
  .container {
    grid-template-columns: 250px 1fr 200px;
    grid-template-areas:
      "header header header"
      "sidebar main aside"
      "footer footer footer";
  }
}

Combining Grid with Flexbox

Grid and Flexbox are complementary technologies. A good rule of thumb:

  • Use Grid for the main page layout - two-dimensional layouts with rows and columns
  • Use Flexbox for UI components - one-dimensional layouts like navigation menus, card components, etc.
/* Overall page layout with Grid */
.page {
  display: grid;
  grid-template-columns: auto 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}

/* Component layout with Flexbox */
.header {
  grid-area: header;
  display: flex;
  justify-content: space-between;
  align-items: center;
}

.nav-menu {
  display: flex;
  gap: 1rem;
}

Browser Support

CSS Grid is now supported in all modern browsers. As of 2023, global support stands at over 95% of browsers in use, making it safe to use in production without fallbacks for most websites.

Practical Examples

Image Gallery

.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  grid-auto-rows: minmax(200px, auto);
  gap: 15px;
}

.gallery-item:nth-child(3n+1) {
  grid-column: span 2;
  grid-row: span 2;
}

Dashboard Layout

.dashboard {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-template-rows: auto auto 1fr;
  grid-template-areas:
    "header header header header"
    "stats stats stats stats"
    "sidebar main main main";
  height: 100vh;
  gap: 20px;
}

.header { grid-area: header; }
.stats {
  grid-area: stats;
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: 10px;
}
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }

Conclusion

CSS Grid has transformed how we approach web layout, making it possible to create complex, responsive designs with clean, readable code. By understanding the fundamentals of Grid, you can create layouts that were previously impossible or required complex hacks.

The beauty of Grid is that it's both powerful and intuitive. The visual nature of the syntax, especially with named areas, makes creating and maintaining layouts much more approachable. Combined with the flexibility offered by features like minmax(), auto-fit, and the fr unit, Grid is the most capable layout system the web has ever had.

When you're ready to take your Grid skills to the next level, explore more advanced topics like subgrids, masonry layouts with Grid, and dynamic layouts with CSS variables.
